Common Linux commands

This page has been created to help users with little to no Linux experience who need help to learn and understand the commands that can be used to navigate Unix systems.

It details some of the most common (and helpful) Linux commands and explains how they can be used.

There is also a list of websites that provide Linux training, some free, some fee-based.

man pages - explanations of commands

The man command (short for manual page) can be run on any Unix software. Man pages are distributed with the software they document and explain what a command does, along with a list of any sub-commands and sub-sub commands that can be used in conjunction.

To read the manual page for any command, type:

man <commandname>

--help : learn more about a command & its features

The help feature can be entered after any command and provides a brief summary of options that can be used in conjunction with your command. All command options are case-sensitive.

<command> --help\n


pwd : see what directory you are in

pwd means 'print name of current/working directory'. If you are unsure what directory you are working from, simply type pwd or:

  • pwd -L (display pathname even if it contains symbolic links)
  • pwd -P (display pathname avoiding all symbolic links)


cd : change the directory you are in

The cd command allows you to change the current directory. The default directory when you access a login node is usually your home directory.

Slurm does not recommend you perform tasks from the home directory, hence you will need to use the cd command to change your working location from the home directory to your lab/project share (or other relevant directories).

cd <directory file path>

eg. cd /stornext/General/data/academic/grp_rdm_test/

mkdir : make a new directory/folder

The mkdir command creates directories (also known as folders) and can also create multiple directories at once.

To make a new directory/folder, simply type mkdir <foldername>

There are also a variety of subcommands that can be used in conjunction with mkdir.

Here are a few:

mkdir CommandMeaningExample
--versionDisplays the version number & some information regarding the licence.mkdir --version
-vDisplays a message for every directory created.mkdir -v one two three
-pA flag that enables the command to create parent directories as necessary. If the directories exist, no error is specified.mkdir -p <foldername>
-vDisplays a message for every directory created.mkdir -v one two three
-p first/second/thirdThis command creates 3 directories.first is created in the working directory
second is created within the first directory
third is created within the second directory

mkdir -v

This is typical of what you will see if you use the mkdir -v command followed by the directory names you wish to create,
eg. 'one', 'two' and 'three'.

You can use any number of directories and any directory names:

-bash-4.2$ mkdir -v one two three\nmkdir: created directory ‘one’\nmkdir: created directory ‘two’\nmkdir: created directory ‘three’\n

As you can see from the text above, the system confirms it has created 3 directories called 'one', 'two' and 'three'.

If there is an error creating one of the directories, a message appears stating the directory could not be created as well as the reason why, eg. mkdir: cannot create directory ‘one’: File exists.

This error message states that directory 'one' could not be created because there is already a folder called 'one'.

In order to successfully continue, either rename the existing folder from 'one' to something new or change the name of the new folder you are trying to create.

ls : list files/folders within your current directory

The ls command lists the files and directories which are located in the working directory. It also has many subcommands that can be used with it, eg. ls -h

Some other examples are:

ls CommandMeaning
-hlists content in human readable format
-luse a long listing format
-mfill width with a comma-separated list of entries
-rlist content in reverse order while sorting
-sdisplay the allocated size of each file, in blocks
-Ssort by file size (largest-smallest)
-1list one file per line


cat : view single & multiple files from the command line

The cat (short for concatenate) command allows us to view the contents of files and/or concatenate files.

If you wish to view a file from the command line, simply type:

cat <filename>

If you wish to see multiple files concatenated from the command line; type the cat command, followed by the different file names you want to view, separated by a space between each file name, eg.:

cat <filename1 filename2 filename3 filename4>

nano : create, view, edit &/or copy files from the command line

The nano command is extremely versatile. It can:

  • create a new file immediately without a name (the name is created upon saving)
  • create a new file with a name
  • add content to the new files
  • open an existing file
  • edit an existing file.

To create a blank file, just type nano at the command prompt and it will open an empty, unnamed shell.

Alternatively, you can create an empty, named file by typing:

nano <filenameofyourchoice>

and it will open your new empty, named file (with the name you dictated at the top of the shell after you typed nano).

The default blank nano screen provides the file name of either:

  • the new named file you have created
  • the existing file you have opened, or
  • ‘new buffer’ indicates the new empty, unnamed shell you have created (yet to be named).

Alternatively, you can open, view, and edit an existing file from the command line by typing:

nano <filename> or nano </filepath/filename>

This will open your file and allow you to edit it. Just start typing and make the changes you need. (The screenshot below shows the file 'foo.sh' open for editing.)
