MaxQuant on SLURM

MaxQuant is a quantitative proteomics software package designed for analysing large mass-spectrometric data sets. It is specifically aimed at high-resolution MS data. Several labelling techniques, as well as label-free quantification, are supported.

The nature article, MaxQuant goes Linux (published 31 May 2018 by multiple authors) shows how MaxQuant is faster (or scales better) on Linux.

Certain phases are also very I/O bound, therefore a very significant improvement can be achieved with the raw files and the fasta file on VAST. If you want to start using VAST, please check the process to request a VAST scratch area here.

Read our MaxQuant step-by-step guide for more information.

You need to update your mqpar.xml config file:

  • Update all raw data file paths to absolute paths, do not use relative paths

  • Update the fasta file paths to absolute paths, do not use relative paths

  • Make sure the number of threads is equal to the number of input files.
  • Make sure the useDotNetCore is set to True.

According to the recommendation in MaxQuant's documentation, the minimum resources required are:

  • 4 GB RAM minimum
  • 4 GB RAM per thread.

You need to use these numbers and update the Slurm job submission job, similar to:

#!/bin/bash\n\n#SBATCH -p regular\n#SBATCH --job-name=MQ\n#SBATCH --time=48:00:00\n#SBATCH --cpus-per-task=36\n#SBATCH --mem-per-cpu=4G\n\n\nmodule load  MaxQuant/2.0.2.0\nMaxQuant mqpar.xml \n
