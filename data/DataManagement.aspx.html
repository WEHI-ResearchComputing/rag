Data management

WEHI has different filesystems that may be managed in different ways to account for all requirements. Some filesystems have quotas set on them like Home areas.

Data tiering is employed to manage the institute's data storage requirements. As the amount of data vastly exceeds the amount of available hard disk, all new or updated data is copied from disk to the tape tier on a nightly basis, thus any file which has been copied to tape may be safely truncated from disk. Files that have previously been truncated may easily be retrieved.​​​​​​​

Store policy

At WEHI the store policies for all managed filesystems are as follows:

  1. Copies are attempted each night. The system cannot guarantee that it will be able to process all the candidate files depending on factors such as the number of files, size of files, and how busy it is with retrieves, as these have a higher priority for resources.
  2. One copy is made to the tape library residing in the Parkville data centre (copy 1) and the second copy is made to the data centre at Melbourne University (copy 2).
  3. If the file is modified this will change the version number. As such the data will be stored as if it's a new file, but associated with the original. Fourteen such copies of the file are kept so that they can be easily reverted to.

    Please note that most editors and programs actually create a new file, copy the data, and delete the original (eg. vim does this by default). Such files will remain at version 1 and the previous one will need to be recovered, rather than revision changed.

If the filesystem becomes 100% full then emergency truncation will occur.

The system needs space to allow for data flow to continue, so it will truncate files at random to get back to its watermark (now set at 95% full) as fast as it can. Go to the Truncation page for more info.

If you have any queries regarding auto-truncation, please contact Research Computing.
