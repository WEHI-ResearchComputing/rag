You may be working at WEHI using a non-WEHI laptop for a number of reasons. If you are, note that you will not be able to use the following:

  • Direct connection to Milton via SSH
  • Connection to the WEHI-staff WiFi network
  • Connection to the GlobalProtect VPN

However, you can still use:

  • OnDemand, which you can use to transfer files to/from Milton, run commands via a shell, run RStudio etc
  • RAP, which you can use to access web services inside the WEHI VPN 
  • Posit Workbench
  • VRA (Virtual Machines)
  • CryoSparc

Getting SSH Access

Since you cannot directly use SSH, you can instead get use a command shell on the Milton HPC in one of two ways:

  • Through OnDemand:
    1. Go to https://ondemand.hpc.wehi.edu.au/ and login

    2. At the top of the page, click “Clusters”, and then “Slurm Shell Access”

  • Through Guacamole
    1. Go to https://rap.wehi.edu.au/http/gpguacamole.wehi.edu.au, and login with your WEHI credentials

    2. Click “SSH - slurm-login” at the bottom of the page 

Accessing OnDemand while On Premises

Although OnDemand is accessible from outside WEHI, if you are physically located in the building and are using one of the public WiFi networks such as eduroam or CPGUEST then OnDemand may not be accessible.

 

If this is the case, and if you have administrator/root access to your non-WEHI machine, you can work around this issue by forcing your machine to use a standard DNS (domain name service) instead of WEHI's internal DNS. To do this, please follow these instructions provided by Google. If asked to choose a WiFi connection, choose the connection you are using at WEHI (eduroam or CPGUEST).

 

If you do not have administrator access to your non-WEHI machine, you will have to work through RAP while on premises, including the use of Guacamole to access a shell (see above).
