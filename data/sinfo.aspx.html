sinfo


The sinfo command can be used to query partition and node status. man sinfo for more detail.

$ sinfo \nPARTITION   AVAIL  TIMELIMIT  NODES  STATE NODELIST \ninteractive    up 1-00:00:00      2    mix milton-med-001,milton-sml-001 \nregular*       up 2-00:00:00     12    mix milton-med-[001-002,004,006],milton-sml-[001-008] \nregular*       up 2-00:00:00      5  alloc milton-lrg-001,milton-med-[003,005,007-008] \nlong           up 14-00:00:0      2    mix milton-med-001,milton-sml-001 \ngpuq           up 2-00:00:00      1   idle milton-gpu-001 \n


To get a summary of each partition, use -s

$ sinfo -s\nPARTITION   AVAIL  TIMELIMIT   NODES(A/I/O/T) NODELIST\ninteractive    up 1-00:00:00          4/0/0/4 milton-med-[001-002],milton-sml-[001-002]\nregular*       up 2-00:00:00        42/0/0/42 milton-lrg-[001-004],milton-med-[001-014],milton-sml-[001-024]\nlong           up 14-00:00:0          2/0/0/2 milton-med-001,milton-sml-001\ngpuq           up 2-00:00:00          0/4/1/5 milton-gpu-[001-005]\n


To get information about a specific partition, use -p

$ sinfo -p gpuq\nPARTITION AVAIL  TIMELIMIT  NODES  STATE NODELIST\nPARTITION AVAIL  TIMELIMIT  NODES  STATE NODELIST\ngpuq         up 2-00:00:00      1 drain* milton-gpu-002\ngpuq         up 2-00:00:00      4   idle milton-gpu-[001,003-005]\n


In the example above, we can see from the STATE column that the nodes are in several different modes.

Node states and their meanings

The table below provides a list of some different node states and their meanings:

Node stateDefinition
DownThe node is unavailable for use
DrainThe node is unavailable for use as per system administrator request
FailThe node is expected to fail soon and is unavailable for use as per system administrator request
MixThe node has some of its CPUs allocated while others are idle
Power_downThe node is currently powered down and is not capable of running any jobs
Power_upThe node is currently in the process of being powered up
RebootThe node is currently scheduled to be rebooted
*The node is presently not responding and will not be allocated any new work. If the node remains non-responsive, it will be placed in the DOWN state (except in the case of COMPLETING, DRAINED, DRAINING, FAIL, FAILING nodes).

To get the features active on each node, use sinfo -o %b,%n

$ sinfo -o %n,%b\nHOSTNAMES,ACTIVE_FEATURES\nmilton-med-001,Broadwell\nmilton-med-002,Broadwell\nmilton-sml-001,Broadwell\nmilton-sml-002,Broadwell\nmilton-sml-015,Broadwell\nmilton-sml-018,Broadwell\nmilton-lrg-001,Skylake\nmilton-lrg-002,Skylake\nmilton-lrg-003,Skylake\nmilton-lrg-004,Skylake\nmilton-med-013,Broadwell\nmilton-med-014,Broadwell\nmilton-sml-003,Broadwell\nmilton-sml-004,Broadwell\nmilton-sml-005,Broadwell\n


On Milton, we have two types of CPU architectures, Skylake, and Broadwell.

For more detailed information about partitions, read the Getting started with SLURM guide.

TIP

If you wanted to ask for the Skylake or Broadwell partition, specify it in your job submit script by writing #SBATCH -C Skylake or in the batch cmd sbatch -C Skylake <script>
