Provisioning and managing interactive VMs

Request a VM for your division, lab, or project

If there is a group of people, for example, a lab or a group of collaborators working on a lengthy project, you can request a VM for this group. This can be provisioned on a long-term basis with the resources you require. It is then up to the members of the group to manage resource use in this VM. Nominated members of the group will have the ability to kill other users' jobs and reboot the system.

Please contact Research Computing to discuss your requirement(s).

The main operating systems and configurations are:

  • Bioinf - Centos 6 (Server): A CentOS 6 (Linux) node that resembles the legacy bioinformatics and population health servers (unix3xx and unix4xx servers). Only use this if you have a dependency on this specific version of CentOS otherwise use the equivalent CentOS 7 template.

  • HPC - Centos 7 (Server): A CentOS 7 (Linux) compute node with a /stornext/HPCScratch/home/<userid> home area (the same as the old Torque compute nodes).

  • SeqMonk: Preinstalled with the Seqmonk program (Java) as a full-screen web app using Guacamole and X windows in a kiosk mode. It has access to the shared filesystems.

  • Structbio - Global Phasing Centos 7: Centos 7 with HPC Scratch home area. Global Phasing structural biology software included. The software will only work with this template.

  • SysBio - MSCypher Windows Server 2012 r2: MSCypher program installed on Windows 2012 r2.

  • HPC - Centos 7 (with VNC): As above but with the same file system as the Structural Biology Unix desktops.

  • WEHI - Windows 7 (Blank): Windows 7 joined to the WEHI Active Directory domain.

  • WEHI - Ubuntu 16: Ubuntu 16.04 Server with the standard Unix home.

Note: Nothing is kept in the VM, all data is saved to the main WEHI filesystems so machines can be destroyed and recreated without losing data, as long as the data is saved to files.

Please consider other alternatives:

  • Work that is not interactive can be run in SLURM batch system.
  • Light interactive work can be run on a shared resource, such as vc7-shared or using interactive batch sessions.
  • Consider requesting a lab or project VM that can be shared between a number of people. Again, please talk to
    Research Computing about your requirements.

    vRA requests - after submission

    Once a vRA request has been submitted, two entries will appear in the deployment list (in most cases):

    1. the actual VM; and
    2. the approval for the VM.

    The entries look similar to the screenshot below:

    • The top entry is the clone.
    • The bottom entry is the approval which can be dismissed (once it is successful) by clicking 'Actions' on the right hand side of the box.

    If the deployment becomes grey and does not proceed, it means your VM is on hold for approval until it has resources to provision.

    Cloud resources

    There are use cases for provisioning resources in an off-premise cloud, for example, specialised images for training. Please discuss your use case with Research Computing.
