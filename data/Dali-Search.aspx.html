Dalilite

Dalilite is a tool for protein structural alignment and structure database search. It is based on the Dali method (Dali stands for distance matrix alignment). It consists of two wrapper scripts:

  • import.pl - convert PDB files to internal data format used by Dali, mirror PDB database
  • dali.pl - perform pairwise structural alignments, all-against-all comparisons, or structural database searches

Dalilite is now available on Milton through the modules system. The installed version supports parallelisation through mpi.

module av dalilite\n\n--------------- /stornext/System/data/modulefiles/structbio/its ----------------\ndalilite/5.1(default)\n\n\n\nmodule help dalilite\n\n----------- Module Specific Help for 'dalilite/5.1' ---------------\n\n    This module sets the environment for dalilite v5.1\n\n    dali.pl -h\n    import.pl -h

To load it: 

module load dalilite\n

import.pl

import.pl converts a PDB entry into the internal data format used by Dali. It reads files in PDB format and creates the corresponding data files, one for each chain, in the data directory. PDB files can be imported one at a time. 

For example:

import.pl --pdbfile <filename> --pdbid <xxxx> [ --dat <path> ]

Options:

      --dat <path>                    directory to store imported data [default: ./DAT]\n        --pdbfile <filename>       PDB formatted file, may be compressed (.gz). <filename> includes path\n        --pdbid <xxxx>                four-letter PDB identifier\n        --pdblist <filename>       list of PDB entries. <filename> includes path and must match the pattern pdb????.ent\n        --rsync                              automated PDB mirroring\n        --pdbmirrordir <path>    PDB mirror directory [default: /data/pdb]\n        --clean                              remove temporary files\n        --verbose                          verbose

dali.pl

dali.pl performs pairwise structural alignment and database searches. The full list of options and syntax can be found here.

For example:

dali.pl  --MPIRUN_EXE mpirun --np 2\t \\\n                                         --cd1 <xxxxx> \\\n                                         --db <filename> \\\n                                         --TITLE <string>  \\\n                                         --dat1 <path>  \\\n                                         --dat2 <path> \\\n                                         --outfmt summary,alignments,equivalences,transrot

Options:   

--cd1 <xxxxX>             query structure identifier       \n--db <filename>          list of target structure identifiers\n\n--dat1 <path>              path to directory containing query data [default: ./DAT/]\n--dat2 <path>             path to directory containing target data [default: ./DAT/]\n\n--title <string>            written to output [default: test]\n\n--outfmt <string>       result blocks to output: summary,alignments,equivalences,transrot [default: summary]\n--clean                         remove temporary files\n\n--np <integer>            number of processes if using openmpi (between 1 and 99) [default: 1]\n


create_dali_summary.py

create_dali_summary.py  is a python script created by Richard Birkinshaw and Julie Iskander that reformats the dali search summary output, adds extra columns, and resaves in csv format. 

The csv file has these 14 columns:

  • No,
  • Chain,
  • Z,
  • rmsd,
  • lali,
  • nres,
  • id,
  • PDB Description,
  • ReferenceProteome,
  • DaliPDBCode,
  • AlphafoldPDB,
  • UniProtID,
  • species,
  • common_name

The file is saved in the path set by option --output_dir or the current directory if that option is not set.

For example:

create_dali_summary.py  --output_dir <path to dali output directory>\n


DaliLite writes a number of intermediate results in the current working directory (CWD). The lock file is deleted automatically if the job is completed successfully. If a file named dali.lock is present, you get the following error message and cannot start another DaliLite job in the same directory:

Directory is locked by dali.lock\n        there may be another DALI process running in this work directory\n         or, the previous run crashed: remove the dali.lock file
