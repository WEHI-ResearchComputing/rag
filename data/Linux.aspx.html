Why Use the Command Line?

Command line is an often-used way to interact with "remote" machines like virtual machines, or supercomputers (HPC), because of the responsiveness and security. It's also great for automation of repetitive tasks (once you get the hang of it). Finally, the command line is useful for developers of code (like researchers!) because developing programs with Graphical User Interfaces (GUIs) take a lot of time!

If you've mostly been using GUI operating systems, like Windows or Mac, and you've got no experience with the Linux command line, this page will help you get started with commands and how to navigate the system.

It details some of the most common (and helpful) Linux commands and explains how they can be used.

For further learning, we recommend this software-carpentry based learning material, which you can work through independantly. We will also occasionally have training that you can attend.

Navigation

Your home directory

When you open a new terminal, or ssh to a directory on Milton or a VM, you will start in your home directory. On my mac, I see the prompt:

mac3953:~ yang.e$

if I'm on Milton, I see:

[yang.e@slurm-login02 ~]$

The prompt is telling me

  1. which machine I'm on. On my Mac, that is mac3953; and on milton, I'm on slurm-login02, one of the "login nodes".
  2. That I'm in the "~" directory on the machine, which is an alias for my home directory.
  3. my username, which is yang.e.

You should see a similar prompt but different machines and operating systems will configure this to appear slightly differently, but will contain mostly the same information.

ls : Seeing what's in your directory

running the ls or "list" command, you'll see the folders and files in your current directory:

mac3953:~ yang.e$ ls\nApplications            Desktop                  Documents \nDownloads               Library                  Movies\nMusic                   OneDrive - wehi.edu.au   Pictures\nPublic

But how do you tell which of these are files and which are directories? You can pass the "-F" option:

mac3953:~ yang.e$ ls -F\nApplications/           Desktop/                 Documents/\nDownloads/              Library/                 Movies/\nMusic/                  OneDrive - wehi.edu.au/  Pictures/ \nPublic/

which adds a forward-slash to the end! This means the objects are all directories. If they were files (like a document), there would be no slash.

You can check the contents of a folder by providing a folder name

mac3953:~ yang.e$ ls -F Downloads\nMicrosoftTeams-image.png\nbiostar-handbook.pdf\n

cd : moving around

The cd command allows you to change your "current working directory". For example, I can change to the "Documents" folder with

mac3953:~ yang.e$ cd Documents\nmac3953:Documents yang.e$\n

You can see that your prompt is updated to reflect your new working directory. I can check the contents of the directory using the ls command used earlier

mac3953:Documents yang.e$ ls -F\nReport.docx\nReceipts/\n

Paths

Paths refer to the location of objects on the filesystem, and can be either relative or absolute. Relative paths are paths that you provide relative to your current working directory. For example,

mac3953:~ yang.e$ cd Documents

Is using a path to the Documents folder relative to my working directory, which was my home directory.

Absolute paths are paths specified from the root directory. You can change to the root directory:

mac3953:~ yang.e$ cd /\nmac3953:/ yang.e$\n

where "/" refers to the root directory and is the directory which contains all other files on the system. Absolute paths begin with the root and describe the full path to the file/directory of interest. For example, the absolute path to the Downloads folder used previously would be /Users/yang.e/Downloads:

mac3953:~ yang.e$ cd /Users/yang.e/Downloads\nmac3953:Downloads yang.e$ \n

pwd : Don't get lost!

It's easy to get lost in the tree of the filesystem, and your prompt may only be showing you your current working directory instead of the full path. To figure out what directory you're in, you can use the pwd command (Present Working Directory):

mac3953:Downloads yang.e$ pwd\n/Users/yang.e/Downloads\n

which tells you the absolute path to your current working directory!

Help is only a command or option away!

Most commands come with a man page (short for manual). These pages give you detailed information on how to use the command. Try learning about the ls command:

mac3953:~ yang.e$ man ls\n

Help

--help : learn more about a command & its features

The help feature can be entered after any command and provides a brief summary of options that can be used in conjunction with your command. All command options are case-sensitive.

<command> --help

ls : Useful options

The ls command lists the files and directories which are located in the working directory. It also has many subcommands that can be used with it, eg. ls -h

Some other examples are:

ls CommandMeaning
-hlists content in human readable format
-luse a long listing format
-mfill width with a comma-separated list of entries
-rlist content in reverse order while sorting
-sdisplay the allocated size of each file, in blocks
-Ssort by file size (largest-smallest)
-1list one file per line


cat : view single & multiple files from the command line

The cat (short for concatenate) command allows us to view the contents of files and/or concatenate files.

If you wish to view a file from the command line, simply type:

cat <filename>

If you wish to see multiple files concatenated from the command line; type the cat command, followed by the different file names you want to view, separated by a space between each file name, eg.:

cat <filename1 filename2 filename3 filename4>

mkdir : make a new directory/folder

The mkdir command creates directories - you just need to pass a path! For example, to create a "thesis" folder in the current working directory:

mac3953:Downloads yang.e$ mkdir thesis\nmac3953:Downloads yang.e$\n

Notice that the command didn't say anything? As a general rule, when using the Linux command line, no news is good news. To make sure, you can check that this new directory was made, with the ls command used earlier.

mac3953:Downloads yang.e$ ls -F\nthesis/\nMicrosoftTeams-image.png\nbiostar-handbook.pdf\n

You can also make multiple directories with one mkdir command by passing multiple paths:

mac3953:Downloads yang.e$ mkdir thesis/references /Users/yang.e/Downloads/thesis/drafts\nmac3953:Downloads yang.e$\n

And you can see that you can mix absolute paths with relative paths too!

Some useful options to use with the mkdir command:

mkdir CommandMeaningExample
--versionDisplays the version number & some information regarding the licence.mkdir --version
-vDisplays a message for every directory created.mkdir -v one two three
-pA flag that enables the command to create parent directories as necessary. If the directories exist, no error is specified.mkdir -p <foldername>
-vDisplays a message for every directory created.mkdir -v one two three
-p first/second/thirdThis command creates 3 directories.first is created in the working directory
second is created within the first directory
third is created within the second directory

mkdir -v

the v is short for verbose and is for when you want to make sure the directories were created. Instead of creating the commands silently, when passed the -v option, mkdir will tell you which directory/ies its created. This is typical of what you will see if you use the mkdir -v command followed by the directory names you wish to create,
eg. 'one', 'two' and 'three'.

-bash-4.2$ mkdir -v one two three\nmkdir: created directory ‘one’\nmkdir: created directory ‘two’\nmkdir: created directory ‘three’\n

As you can see from the text above, the system confirms it has created 3 directories called 'one', 'two' and 'three'.

Text editing

Two common command line text editors are Vim and nano.

nano

is pretty easy to learn and a good first-choice. To open a file with nano, use the command

nano name-of-file

If "name-of-file" doesn't exist, nano will create the file. Running this command will open a window like

  UW PICO 5.09                             File: newfile.txt                                \n\n\n\n\n\n\n\n\n^G Get Help    ^O WriteOut    ^R Read File   ^Y Prev Pg     ^K Cut Text    ^C Cur Pos       \n^X Exit        ^J Justify     ^W Where is    ^V Next Pg     ^U UnCut Text  ^T To Spell

With a blinking cursor near the top-left of the window. You can start typing and text will appear - just like text editors or Word. The instructions at the bottom of the window are the commands, where "^" means "Ctrl" e.g., Pressing Ctrl+G shows you the help page for nano.

To save what you write in nano, you can press Ctrl+O (WriteOut) to write the content to the file. If you're writing to a new file, the bottom bar may prompt you. For example, if I press Ctrl+O after writing "demo text":

  UW PICO 5.09                             File: newfile.txt                    Modified  \n  \ndemo text\n\n\n\nFile Name to write : newfile.txt                                                \n^G Get Help  ^T  To Files                                                     \n^C Cancel    TAB Complete

The bottom changes and is prompting you for more information. If I press Enter, it will save the text to newfile.txt and then return to the same window above, where you can continue editing.

To save and close, you can press Ctrl+X. The bottom text will then change will prompt you to confirm whether you want to save the changes.

Save modified buffer (ANSWERING "No" WILL DESTROY CHANGES) ?                            \n              Y Yes                                                                 \n^C Cancel     N No

Press Y (without Ctrl) to save confirm that you wish to save, or press N to close without saving. If you wish to save, you will need to press Enter again to confirm that you want to write the content to the file you specified.

vim

Is often used as the default editor in applications (such as git). However, it takes a little practice to get used to! We generally recommend starting with nano, but it's useful to know how to do the basics in vim because it's often set as the default editor.

Like nano, you can open or create a new file by passing the file name as an argument to the vim command:

vim anotherfile.txt
~                                                              \n~                                                              \n~                                                              \n~                                                              \n"anotherfile.txt" [New File]

But unlike nano, you can't edit straightaway! To begin editing, press "i" (i for insert) and you can type away! Once you've finished writing your text, you can press "Esc", which takes you back to "normal" mode. In normal mode, type ":wq" to save and quit. If you wish to quit and discard changes, you can type ":q!" instead.

You can go through the inbuilt tutorial but running the command "vimtutor":

$ vimtutor\n\n\n===============================================================================\n=    W e l c o m e   t o   t h e   V I M   T u t o r    -    Version 1.7      =\n===============================================================================\n\n     Vim is a very powerful editor that has many commands, too many to\n     explain in a tutor such as this.  This tutor is designed to describe\n     enough of the commands that you will be able to easily use Vim as\n     an all-purpose editor.\n...\n

rm : remove

rm is used to delete files and has multiple subcommands to enhance its use and enable it to delete directories. Deleting files/directories is an irreversible operation, make sure that you are deleting the right file/directory.


To delete a single file: rm <filename>

To delete multiple files at once: rm <file1> <file2> <file3> <file4>

Some other examples include:

rm CommandMeaning
-f
--force		Ignore non-existent files & arguments, never prompt
-iPrompt before every removal
-lPrompt once before removing more than three files, or when removing recursively (less intrusive than -i while still giving protection against mistakes)
-r
-R
--recursive		Remove directories and their contents recursively
-d <foldername>
--dir <foldername>		Remove empty directories
-v
--verbose		Explain what is being done


Prompt before deleting a file by adding -i after rm:

rm -i <filename>

-bash-4.2$ rm -i foo3.sh\nrm: remove regular file ‘foo3.sh’? y\n-bash-4.2$\n


To remove a directory and its subdirectories interactively, just add the parent directory name after the command string:

rm -ri <parentfolder>

-bash-4.2$ rm -ri first\nrm: descend into directory ‘first’? y\nrm: descend into directory ‘first/second’? y\nrm: remove directory ‘first/second/third’? y\nrm: remove directory ‘first/second’? y\nrm: remove directory ‘first’? y\n-bash-4.2$\n\n


rmdir : remove directory

As the name suggests, rm removes a directory
(rm is designed to remove files, unless paired with a subcommand).

Type:

rmdir <directoryname>

history : display commands previously used on a login node

If you have forgotten the jobs you ran or commands you used on a login node, type history and the system will return a list of all commands you have entered. This allows you to go back and refresh your memory.

history CommandMeaning
historydisplays command line history with line numbers (lines listed with * have been modified)
-cclears the history list by deleting all the entries
-d <offset>delete the history at position offset
<number>lists the last xx number of lines in the history


logout : exit the login shell

You can type exit or logout to close the connection to your login shell.
