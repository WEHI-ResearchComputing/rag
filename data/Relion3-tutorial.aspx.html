Relion 3 - WEHI tutorial

This page will help you to run this Relion 3 tutorial. The dataset used EMPIAR-10204, beta-Galactosidase (JEOL CryoARM 200 kV microscope with a Gatan K2 camera).

The tutorial movies can be found:

/stornext/Projects/cryoEM/cryoEM_data/tutorial_data/relion30_tutorial_data.tar\n

Pre-calculated results are in:

/stornext/Projects/cryoEM/cryoEM_data/tutorial_data/relion31_tutorial_precalculated_results.tar.gz.\n\n

Start Relion 3

From a terminal window, log in to vc7-shared:

ssh -Y vc7-shared\n

You can also run Relion through an interactive session on Slurm.

Note: If you are using a mac laptop, make sure you install XQuartz before you log in.

In your workspace (VAST scratch is recommended), create a new folder and open it:

mkdir relion_tutorial && cd relion_tutorial\n

Decompress the dataset and the precalculated results in your folder:

tar xf /stornext/Projects/cryoEM/cryoEM_data/tutorial_data/relion31_tutorial_precalculated_results.tar.gz\n\ntar xf /stornext/Projects/cryoEM/cryoEM_data/tutorial_data/relion30_tutorial_data.tar\n

Copy mtf_k2_200kV.star from the pre-calculated results to the tutorial directory:

cp PrecalculatedResults/mtf_k2_200kV.star relion30_tutorial/\n

Load the required Relion module:

module use /stornext/System/data/modulefiles/nvidia \nmodule load relion/3.1beta_1cu10.1\n

Navigate to the relion30_tutorial folder:

cd relion30_tutorial/\n

Start Relion:

relion \n

You will receive a message similar to the following:

[1] 20671\n[relion30_tutorial]$  Only run the relion GUI from your ProjectDirectory. Do you want to start a new project here [y/n]?\n

Press y, then the Relion GUI will appear, as shown below. It may take a few minutes to appear.

Motion Correction

CPU-only RELION implementation

This job takes approximately 6 minutes.

The slurm submission script, used in the Running tab (standard submission script), can be found at​ /stornext/System/data/nvidia/relion/.

You have to use the slurm batch script at:

/stornext/System/data/nvidia/relion/sbatch3.bash​​​​​​​

GPU-based MotionCor2

To use GPU based motion correct (UCSF MotionCor2), you need to load the following module before starting Relion:

module load motioncor2/1.3.1cu10.1\n

Remember that if your job will need more time than the limit allows for slurm jobs (2 days), you can email the Help Desk or Research Computing.
