​​​​​Posit Workbench

As discussed in the R Services page, Posit Workbench is one of three methods you can use to access R at WEHI. Please refer to this page for general R usage, including R library management. Note that RStudio OnDemand is significantly preferred to Posit Workbench.

This page describes some tips about using Posit Workbench, that do not apply to the other methods of accessing R.

Note: Posit Workbench is accessible directly with/without VPN.​​​​

Note: After 6th April 2023, Posit Workbench will require a VAST Scratch space to use. Get VAST Scratch by completing this form​​​​​​​

Sessions

Unlike RStudio OnDemand, Posit Workbench allows users to work in multiple sessions at a time. The Posit Workbench Sessions page details every open session and/or project and is where to:

  • view session details
  • quit a session
  • suspend a session; and
  • open a new session.

Accessing the sessions page

 

​​

 

View session details

If you have notified the WEHI Helpdesk about a session that has hung or frozen, they may ask you to send them the session's details.

Go to the Sessions page and click the 'info' button on the right-hand side of the session.

 

The session details window looks like this image on the right:

Remember to obtain the session details prior to quitting a session because once the session has quit, it is no longer possible to obtain its details.

​​​​​​​NB:  If you have multiple sessions open and are unsure which session you want to terminate, 
click the blue title to enter the session and check it is the correct one.
To return to the Sessions page, simply click the back button.

Quit a session

Occasionally an active Posit Workbench session will fail to load, hang or freeze. When this happens, quit the frozen session, then open a new session in order to continue with your work.

Click the Quit button on the right side of a session and it will terminate immediately (as shown in the image on the left).

It is also possible to quit all active sessions at once by clicking the Quit all button at the top of the Sessions page.

Open a new session

To start a new session, from the Sessions page, click the button at the top of the page.

Errors

On occasion, Posit Workbench may produce an error and there is one known error that occurs infrequently.

After the login details are entered, instead of the usual home screen appearing, the error may cause:

  • an empty screen
  • the home page not to load
  • no response
  • an inability to log in.

If this (or any other error) occurs whilst you are using RStudio, please contact the Helpdesk.

Orphaned Sessions Issue

If you encounter an issue where your memory usage on the RStudio dashboard is unusually high, but none of your open sessions are using much RAM, you might be the victim of the orphaned sessions bug. To understand and resolve this bug, refer to the following newsletter article.​​​​​​​

Usage Policies

In order to maintain the stability of the Posit Workbench, it is currently subject to the following usage policies:

  • A limit of 3 sessions per user
  • Each user cannot use more than 24 GB of RAM across all their sessions. For more information on the latest user limits please refer to this page. Hitting this cgroup memory limit may result in your RStudio sessions being automatically killed or restarted, or the error message "Error: vector memory exhausted (limit reached?)".
  • Sessions that are idle for more than 3 hours will be automatically suspended to disk. Idle time is the amount of time since a session has run any R commands, including commands typed by the user in the console and the execution of any R scripts. Suspended sessions retain all your variables as they were in the session, but start-up time will be longer when loading a suspended session.
  • Due to sessions being saved to VAST Scratch, unused sessions will be cleaned up after 2 weeks. This will delete variables in the environment of your session. So please save your results if you aren't using your project for more than a week!
