Setup Authentication With SSH

GitHub no longer allows command-line access to accounts with username and password. Instead, we will create an SSH key. We will go through the steps for Linux (Milton) but they are similar on macOS and windows - https://docs.github.com/en/authentication/connecting-to-github-with-ssh/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent.

The following instructions are performed on Milton. After SSH'ing to Milton:

1. Create an SSH key pair

Execute the "ssh-keygen" command like so:

ssh​​​​​​​-keygen -t ed25519 -C "your_git_email@example.com"\n

When you’re prompted to “Enter a file in which to save the key,” press Enter. This accepts the default file location. Here, the default key name is id_ed25519. You can change the path and name of the key from this but make sure you change it in the steps below as well

At the prompt, type a secure passphrase.

This will create two new files in ~/.ssh: id_ed25519 and id_ed25519.pub. The first one is your private key, and the second is the public key. Make sure not to share your private key!

For example:

[bollands.c@slurm-login02 git_example]$ ssh-keygen -t ed25519 -C cgbollands@gmail.com\nGenerating public/private ed25519 key pair.\nEnter file in which to save the key (/home/users/allstaff/bollands.c/.ssh/ied_ed25519):\nEnter passphrase (empty for no passphrase):\nEnter same passphrase again:\nYour identification has been saved in /home/users/allstaff/bollands.c/.ssh/id_ed25519.\nYour public key has been saved in /home/users/allstaff/bollands.c/.ssh/id_ed25519.pub.\nThe key fingerprint is:\nSHA256:WXGoyliRD7TRIQaTuZ8XjD5YD4cyqGFsb69dW7moiI  cgbollands@gmail.com\nThe key's randomart image is:\n+--[ED25519 256]--+\n|  .o   =o ...    |\n|  ..* X . .o     |\n|  .X % O ..      |\n| .. B % =o       |\n|  .  B BS        |\n|   o. =          |\n|  . . . .        |\n|E .  o.oo        |\n| . .o. =o        |\n+----[SHA256]-----+\n

3. Start an "SSH agent"

An SSH agent will use your keys to authenticate

[bollands.c@slurm-login02 git_example]$ eval $(ssh-agent)\nAgent pid 678\n

4. Add your private key to the SSH agent

[bollands.c@slurm-login02 git_example]$ ssh-add ~/.ssh/id_ed25519\nIdentity added: /home/users/allstaff/bollands.c/.ssh/id_ed25519 (cgbollands@gmail.com)\n

5. Get the public key contents

The public key is the key with the ".pub" extension. You can dump the contents of the key to your terminal with "cat". Copy the contents (highlight and Ctrl/Command+C)

[bollands.c@slurm-login02 git_example]$ cat ~/.ssh/id_ed25519.pub\nssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIG8+SHwzfT8+Bw0Ki22l4EKzd98gKY1g7O6yo9r2oxyE cgbollands@gmail.com\n

6. Add the public key to your GitHub account

Open GitHub and go to your account settings -> Access -> SSH and GPG keys.

Select add new SSH key

Create a descriptive name and paste the key into the text box supplied.


Creating a Repository on GitHub

1. Select the green button to create a new repository

2. Create a meaningful repository name and optionally add a description

The name must be unique within your or your organisation GitHub. The description can always be added later.

3. Choose whether your repository is private or public

By default, the repo is set to private, it is recommended to keep it as that unless you require people outside of the organisation to be able to view the repo. You can always make the repo public later on.

4. (Optional) Add a README

If you wish to add a written description within the repo for people viewing it on GitHub, select “Add a README file” this can be updated later.

5. Create the repository

Click on "Create"

"Cloning" the GitHub repository onto your computer

Once your repository is created on GitHub, it's ready to be "cloned" which enables you to edit and update the contents locally!

Get the SSH link by from your repository by clicking on the green "Code" button:

Note that the default link will use the HTTPS protocol. To ensure you can edit the repository, you should use the "SSH" link like in the above picture.

You can then clone the repository with the link from your command line. For example:

git clone git@github.com:WEHI-ResearchComputing/git_example.git

Making Changes to the Repository

After cloning your GitHub repository, start your workflow by creating a test file with some text in it via the command line. To add the file to your repo type the following:

git add .

This:

  • Stages all the new content in the current directory to prepare for the next commit.
  • The “.” indicates that you are adding all the files that are present whether you have changed them or not, git determines the changes and adds the necessary files.
  • You can add specific files by giving the file location instead of .. For example: "​​​​​​​​​​​​​​git add git_test/test.csv"
git commit -m “<add message about this commit here>”
  • "git commit" creates a new commit to the repository with the contents of the previously added files
  • -m creates a message, if you do not add anything a file will open to add one anyway. A good commit message is succinct and completes the sentence: “Once applied, this commit will…”
git push
  • push the changes that have been committed to the GitHub copy of the repository.

Updating the Local Repository With Changes

If changes have been made to the GitHub version of the repository, that you would like to get on your local machine, you run:

git pull

Example output looks like:
