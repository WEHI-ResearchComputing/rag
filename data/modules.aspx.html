Accessing software

Modules system

All application software on Milton nodes is provided through the modules system: modules.sourceforge.net/

The modules system configures the user's unix environment to run specific applications, or indeed specific versions of applications, and to avoid conflicts from mutually exclusive software. The unix environment here refers to environment variables​ such as the ubiquitous $PATH, and also application-specific environment variables such as $PYTHONPATH.

The modules system is accessed via the module​ command. To list available software, log in to the unix system you wish to use:

This list is an example only. New software is being added on almost a daily basis and the precise list of available software will vary from day to day. The subcommand av is short for avail which can also be used.

A module is then loaded to make it available for use:

$ module load samtools\n

A specific version can also be loaded if required:

$ module load samtools/1.4.1\n

If no version is specified then a default is chosen as indicated in the available software list (usually the latest version). If there is a conflict you can remove the module from your environment:

$ module load samtools/1.4.1\n

If no version is specified then a default is chosen as indicated in the available software list (usually, but not always, the latest version). If there is a conflict you can remove the module from your environment:

$ module unload samtools\n

To unload all loaded modules:

$ module purge\n

You can see the modules you have loaded:

$ module list\nCurrently Loaded Modulefiles:\n  1) samtools/1.5    2) varscan/2.3.9   3) R/3.4.0\n

You can see how a particular module will modify the environment:

Module configuration files can reside in a number of directories, for example, there are module directories for bioinformatics software and module directories for structural biology software. You may need to access modules in additional directories and you can instruct the module system to use those directories as follows:

$ module use /stornext/System/data/modulefiles/bioinf/its\n

When submitting batch jobs, your job script may need to issue some module commands so that the software required by the batch job is available. Module commands can also be included in the .bashrc so that your environment does not need to set up each time you log in.
