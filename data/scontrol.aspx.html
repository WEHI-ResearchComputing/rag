scontrol

The scontrol command obtains detailed information about jobs or SLURM configurations

To obtain detailed information about a specific job, type:

scontrol show job <jobid>\n\n
$ scontrol show job 1289022\nJobId=1289022 JobName=sys/dashboard/sys/Jupyter\n   UserId=iskander.j(4165) GroupId=allstaff(10908) MCS_label=N/A\n   Priority=423 Nice=0 Account=wehi QOS=normal\n   JobState=RUNNING Reason=None Dependency=(null)\n   Requeue=1 Restarts=0 BatchFlag=1 Reboot=0 ExitCode=0:0\n   RunTime=06:34:30 TimeLimit=08:00:00 TimeMin=N/A\n   SubmitTime=2021-03-12T08:43:41 EligibleTime=2021-03-12T08:43:41\n   AccrueTime=2021-03-12T08:43:41\n   StartTime=2021-03-12T08:43:41 EndTime=2021-03-12T16:43:41 Deadline=N/A\n   SuspendTime=None SecsPreSuspend=0 LastSchedEval=2021-03-12T08:43:41\n   Partition=regular AllocNode:Sid=ondemand:19361\n   ReqNodeList=(null) ExcNodeList=(null)\n   NodeList=milton-sml-020\n   BatchHost=milton-sml-020\n   NumNodes=1 NumCPUs=4 NumTasks=1 CPUs/Task=4 ReqB:S:C:T=0:0:*:*\n   TRES=cpu=4,mem=10G,node=1,billing=24\n   Socks/Node=* NtasksPerN:B:S:C=0:0:*:1 CoreSpec=*\n   MinCPUsNode=4 MinMemoryNode=10G MinTmpDiskNode=0\n   Features=(null) DelayBoot=00:00:00\n   OverSubscribe=OK Contiguous=0 Licenses=(null) Network=(null)\n   Command=(null)\n   WorkDir=/home/users/allstaff/\n   StdErr=/home/users/allstaff/\n   StdIn=/dev/null\n   StdOut=/home/users/allstaff/\n   Power=\n   MailUser=iskander.j MailType=BEGIN\n


As you can see from the example above, there are a number of results provided, including:

  • job id & job name
  • job priority
  • job state (completed, failed, running etc)
  • job run time
  • time job submitted
  • job start/end time
  • partition name
  • number of tasks running (NumCPUs).


To show partition details, type:

scontrol show partition <partition_name>\n
