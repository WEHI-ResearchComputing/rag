squeue

The squeue command is used to query jobs currently running or pending not completed in the queue.

-j <jobID> displays info on a specific job:

$ squeue -j 1288143\n        JOBID PARTITION     NAME     USER ST       TIME  NODES NODELIST(REASON)\n        1288143      gpuq cryospar cryospar  R    1:08:45      1 milton-gpu-001\n


To show the expected start time of a pending job:

$ squeue -j <jobID> --start\n


-u <username> displays jobs in the queue for a specific user:

$ squeue -u cryosparc\n        JOBID PARTITION     NAME     USER ST       TIME  NODES NODELIST(REASON)\n        1288143      gpuq cryospar cryospar  R    1:08:45      1 milton-gpu-001\n


You can also customize the output of the squeue and sinfo commands. Many fields are available that aren’t shown in the default output format. For example, we can add a TIME_LEFT field which says how much time is left before the job’s wall time limit is reached.

The example below shows how you can adapt the squeue command to obtain more detailed information.

-bash-4.2$ squeue --format '%.7i %.9P %.8j %.8u %.2t %.10M %.6D %.8h %.12L %R'\nJOBID PARTITION     NAME   USER    ST     TIME   NODES OVER_SUB   TIME_LEFT NODELIST(REASON)\n10346   gpuq     Class2D/  james.s  R    2:07:58   1       OK     21:52:02  SLURM-gpu-n02\n10344   gpuq     Class2D/  james.s  R    7:31:44   1       OK     20:28:16  SLURM-gpu-n01\n10343   gpuq     Class2D/  james.s  R    7:32:11   1       OK     20:27:49  SLURM-gpu-n04\n


We’ve specified %.12L, (for TIME_LEFT) in addition to some other standard fields, to obtain this output.

For all available fields and other output options, see the squeue and sinfo man pages, or refer to the SLURM documentation.
